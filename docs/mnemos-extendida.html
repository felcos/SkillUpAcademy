Busqueda plana

Motor 1 — Retencion

Motor Bjork

El motor central de MNEMOS. Modela como recordamos y olvidamos usando la New Theory of DisuseBjork & Bjork, 1992La teoria propone dos fuerzas independientes: Storage Strength (permanencia) y Retrieval Strength (accesibilidad). RS decae con el tiempo, SS NUNCA decrece. Cuanto mas bajo RS al reactivar, mayor el incremento en SS (desirable difficulty).«The act of retrieval is itself a powerful learning event.»Bjork, R.A. & Bjork, E.L. (1992). A New Theory of Disuse. de Robert y Elizabeth Bjork.

Formulas matematicas

Decaimiento de RS (lazy): RS(t) = RS₀ × e-(t - tlast) / λ donde λ = 86400 × SS

Incremento de SS: ΔSS = α × (1 - RSactual) donde α = 0.3

Invariante absoluto: SS NUNCA decrece. Si ΔSS < 0, se clampea a 0.

Ejemplo numerico

Un memor con SS=2.0, RS=1.0 creado hace 48 horas:

Lambda = 86400 * 2.0 = 172800 segundos (2 dias)
Elapsed = 48h = 172800 segundos
RS(48h) = 1.0 * e^(-172800/172800) = 1.0 * e^(-1) = 0.368

Si lo recuperas ahora:
  DeltaSS = 0.3 * (1 - 0.368) = 0.3 * 0.632 = 0.190
  New SS = 2.0 + 0.190 = 2.190  (SUBIO - nunca baja)
  New RS = 1.0  (reset por recuperacion)
  New Lambda = 86400 * 2.190 = 189216 seg (decae mas lento ahora)
BjorkEngine.CalculateRSAvanzado: Codificacion
public static float CalculateRS(Memor memor, DateTime now)
{
    if (now <= memor.RsUpdatedAt)
        return memor.RetrievalStrength;

    float elapsedSeconds =
        (float)(now - memor.RsUpdatedAt).TotalSeconds;
    float lambda = memor.LambdaDecay;

    if (lambda <= 0f) return 0f;

    float decay = MathF.Exp(-elapsedSeconds / lambda);
    return memor.RetrievalStrength * decay;
}
El calculo es LAZY: RS no se actualiza en la BD continuamente. Se computa on-demand cuando alguien accede al memor. BjorkEngine.UpdateOnRetrievalAvanzado: Codificacion
public static BjorkUpdate UpdateOnRetrieval(
    Memor memor, DateTime now)
{
    float currentRS = CalculateRS(memor, now);
    // Desirable difficulty: bajo RS = alto deltaSS
    float deltaSS = Alpha * (1f - currentRS);
    if (deltaSS < 0f) deltaSS = 0f; // INVARIANTE

    float newSS = memor.StorageStrength + deltaSS;
    float newLambda = BaseLambdaMultiplier * newSS;

    return new BjorkUpdate {
        NewSS = newSS,
        NewRS = 1.0f, // Siempre reset
        NewLambdaDecay = newLambda
    };
}

Archival StatusAvanzado: FuncionamientoBjorkEngine clasifica memors en 4 niveles segun RS actual:

RS > 0.5 = Active (acceso facil)
RS 0.2-0.5 = Warm (aun accesible)
RS 0.05-0.2 = Cold (dificil acceso)
RS < 0.05 = Archived (casi inaccesible)

Esto NO borra memors. Solo cambia su ArchivalStatus para que el Consolidator pueda priorizar. El SS sigue intacto.

Los memors se clasifican automaticamente en Active/Warm/Cold/Archived segun su RS actual. Esto permite al ConsolidatorAgent priorizar que memors revisar.

Bjork se conecta con CRR y el BusAvanzado: IntegracionCon CRR: RS alimenta el alpha2 (peso 0.11) del Cognitive Retrieval Ranking. Memors con RS alto se rankean mejor.

Con Bus Cognitivo: Cuando RS cae por debajo de un umbral critico, se emite RetrievalStrengthCritical. El handler activa el memor en el SAN para boostearlo en busquedas posteriores.

Con Consolidator: El agente usa CalculateRS para recalcular RS de memors en batch. Si RS < 0.05 durante consolidacion, propone archival.

H2: Bjork predice accesibilidad real. Medible: comparar RS predicho vs. tiempo real de recuperacion del usuario. Si correlacion > 0.6, la hipotesis se sostiene.

Motor 3 — Certeza

Motor Bayesian

Rastrea tus creencias con credence dualEpistemologia BayesianaEl teorema de Bayes permite actualizar creencias ante nueva evidencia. MNEMOS implementa DOS credencias independientes: credence_ideal (lo que la evidencia dice) y credence_actual (lo que realmente crees). La diferencia es el bias_gap.Bayes, T. (1763). An Essay towards solving a Problem in the Doctrine of Chances. Phil. Trans. Royal Society, 53. : lo que la evidencia dice vs. lo que realmente crees.

Credence dual

I

credence_ideal

Lo que DEBERIA creer segun la evidencia. Se actualiza automaticamente via BayesianEngine cuando se registra nueva evidencia. El usuario NO lo controla directamente.

A

credence_actual

Lo que REALMENTE cree el usuario. Solo el usuario puede cambiarlo. El sistema puede sugerir, nunca imponer.

G

bias_gap

|credence_actual - credence_ideal|. Si ≥ 0.15, se detecta una divergenciaAvanzado: FuncionamientoCuando el bias_gap supera 0.15, BayesianEngine clasifica la divergencia en 6 tipos:

1. Confirmation: actual > ideal, tiene revisiones previas
2. Anchoring: actual > ideal, sin revisiones (anclado al valor inicial)
3. DunningKruger: actual > 0.9 pero ideal < 0.7
4. Recency: actual < ideal, muchas revisiones
5. Availability: actual < ideal, pocas revisiones
6. SunkCost: creencia antigua con resistencia al cambio

El usuario tiene 3 opciones: CorrectToBayesian, KeepAsPreference, o ExploreFurther. .

Formula de actualizacion

Update simplificado:
Si evidencia Supporting: rawDelta = evidenceStrength × (1 - credence_ideal)
Si evidencia Contradicting: rawDelta = -evidenceStrength × credence_ideal
dampenedDelta = rawDelta × (1 - entrenchment × 0.7)
Clampeado a [-0.3, +0.3] (MaxSingleUpdateDelta)
newCredenceIdeal = clamp(credence_ideal + dampenedDelta, 0, 1)

public static BayesianUpdate CalculateUpdate(
    Belief belief,
    EvidenceDirection direction,
    float evidenceStrength)
{
    if (direction == EvidenceDirection.Neutral)
        return unchanged;

    float rawDelta = direction == Supporting
        ? evidenceStrength * (1f - belief.CredenceIdeal)
        : -evidenceStrength * belief.CredenceIdeal;

    float entrenchmentDamping =
        1f - (belief.EntrenchmentScore
              * EntrenchmentDampingFactor);
    float dampenedDelta =
        rawDelta * entrenchmentDamping;
    dampenedDelta = Math.Clamp(
        dampenedDelta, -0.3f, 0.3f);

    float newCredenceIdeal = Math.Clamp(
        belief.CredenceIdeal + dampenedDelta, 0f, 1f);
    // ... return BayesianUpdate
}

public static DivergenceDetection? DetectDivergence(
    Belief belief)
{
    float biasGap = belief.BiasGap;
    float absBiasGap = MathF.Abs(biasGap);
    if (absBiasGap < DivergenceThreshold)
        return null; // No hay divergencia

    DivergenceType type =
        ClassifyDivergence(belief, biasGap);
    float severity = Math.Clamp(
        (absBiasGap - 0.15f) / (1f - 0.15f),
        0f, 1f);

    return new DivergenceDetection {
        BiasGap = biasGap,
        DivergenceType = type,
        Severity = severity
    };
}

Motor 4 — Decision

Motor Kahneman

Rastrea decisiones, detecta patrones recurrentes, y clasifica pensamiento en Sistema 1 vs Sistema 2Daniel Kahneman (1934-2024)Premio Nobel 2002. Sistema 1: pensamiento rapido e intuitivo. Sistema 2: lento y deliberado. Con experiencia, patrones migran de S2 a S1 (automatizacion). Si un patron S1 empieza a fallar, vuelve a S2.Kahneman, D. (2011). Thinking, Fast and Slow. Farrar, Straus and Giroux..

Decisiones inmutables

Una vez creada, la Decision es inmutable — solo los campos de outcome se actualizan al resolverse. Description, PredictedOutcome y ReasoningSystem nunca cambian despues de la creacion.

Patrones y System Transitions

TransicionCondicion
System2 → TransitioningSuccessRate ≥ 0.8 AND TimesApplied ≥ 10
Transitioning → System1SuccessRate ≥ 0.8 AND TimesApplied ≥ 20
System1 → System2SuccessRate < 0.5 (patron fallando)

public static DecisionResolution ResolveDecision(
    Decision decision,
    string actualOutcome,
    float outcomeSatisfaction)
{
    if (decision.IsResolved)
        throw new InvalidOperationException(

    return new DecisionResolution {
        DecisionId = decision.Id,
        ActualOutcome = actualOutcome,
        OutcomeSatisfaction = outcomeSatisfaction,
        IsSuccess = isSuccess,

}
 KahnemanEngine.DetectSystemTransitionAvanzado: Codificacion
public static ReasoningSystem? DetectSystemTransition(
    DecisionPattern pattern)
{
    if (pattern.TimesApplied < MinPatternApplications)
        return null; // Pocos datos

    // Alto exito + muchas aplicaciones = automatico
    if (pattern.SuccessRate >= 0.8f
        && pattern.TimesApplied >= 10
        && pattern.SystemType ==
           ReasoningSystem.System2Deliberate)
        return ReasoningSystem.Transitioning;

    // Exito sostenido = completamente S1
    if (pattern.SuccessRate >= 0.8f
        && pattern.TimesApplied >= 20
        && pattern.SystemType ==
           ReasoningSystem.Transitioning)
        return ReasoningSystem.System1Intuitive;

    // S1 fallando = volver a S2
    if (pattern.SuccessRate < 0.5f
        && pattern.SystemType ==
           ReasoningSystem.System1Intuitive)
        return ReasoningSystem.System2Deliberate;

    return null;
}

La Innovacion Central

CRR v5 — Cognitive Retrieval Ranking

No busca texto similar — busca lo que es cognitivamente relevante para ti ahora. Combina 9 dimensiones en una formula ponderada unica en la industria.

Formula completa

CRR = 0.22·Semantic + 0.10·Text + 0.11·RS + 0.11·Evidence + 0.07·Credence + 0.07·Outcomes + 0.14·Context + 0.09·Serendipity + 0.09·Activation

Red de Activacion Semantica

Spreading Activation Network

Basado en Collins & Loftus (1975)Spreading Activation TheoryCollins y Loftus propusieron que los conceptos en memoria estan organizados en una red donde la activacion se propaga de un nodo a sus vecinos. Cuando piensas en «perro», se activan «gato», «ladrar», «mascota» automaticamente. MNEMOS implementa esto como un servicio in-memory singleton.Collins, A.M. & Loftus, E.F. (1975). A spreading-activation theory of semantic processing. Psychological Review, 82(6). — cuando accedes a un concepto, conceptos relacionados se activan automaticamente.

Decaimiento lazyAvanzado: Codificacion
private static float ComputeDecayedActivation(
    ActivatedNode node, DateTime? now = null)
{
    DateTime currentTime = now ?? DateTime.UtcNow;
    double elapsedSeconds =
        (currentTime - node.ActivatedAt).TotalSeconds;
    if (elapsedSeconds <= 0)
        return node.ActivationLevel;

    float decayed = node.ActivationLevel
        * MathF.Exp((float)(-elapsedSeconds / Tau));
    return decayed;
}

    float dot = 0f, normA = 0f, normB = 0f;
