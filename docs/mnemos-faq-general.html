Publico Objetivo

Quien usa MNEMOS?

MNEMOS esta disenado para cualquier persona cuyo trabajo dependa de acumular, conectar y recuperar conocimiento a lo largo del tiempo. No es una app de notas para listas de la compra — es un sistema para quien necesita que su conocimiento evolucione con ellosCiencia CognitivaLa teoria constructivista de Piaget (1952) establece que el conocimiento no se acumula pasivamente sino que se reorganiza activamente mediante asimilacion y acomodacion. MNEMOS implementa este ciclo directamente en su motor de schemas.Piaget, J. (1952). The Origins of Intelligence in Children. International Universities Press..

I

Investigador

Lee cientos de papers, necesita conectar hallazgos entre disciplinas. MNEMOS vincula memors a schemas y detecta conexiones que el investigador no veria manualmente.

E

Ejecutivo

Toma decisiones estrategicas basadas en datos dispersos. El motor Kahneman rastrea patrones de decision, outcomes y sesgos cognitivos (System 1 vs System 2).

S

Estudiante

Necesita retener y conectar conceptos. El motor Bjork modela la curva de olvido real y prioriza lo que necesita refuerzo mediante desirable difficulty.

D

Desarrollador

Acumula decisiones tecnicas, patrones, bugs resueltos. MNEMOS conecta soluciones pasadas con problemas actuales via CRR (busqueda cognitiva, no textual).

K

Knowledge Worker

Consultores, abogados, medicos — cualquiera que necesite recordar precedentes, conectar casos y evolucionar su criterio profesional con el tiempo.

Modelo de Negocio y Licencia

Cuanto cuesta MNEMOS?

MNEMOS usa un modelo open coreNota TecnicaOpen core significa que el nucleo es open source y gratuito, mientras que funcionalidades avanzadas tienen licencia comercial. Ejemplos exitosos: GitLab (MIT core + Enterprise), Elastic (Apache 2.0 core + comercial), Grafana (AGPL core + Enterprise). con 4 capas de licencia. El nucleo completo (4 motores cognitivos, CRR, API, MCP) es gratuito y open source.

El 100% de la ciencia cognitiva es gratuita. Los motores Bjork, Piaget, Bayesian y Kahneman, el CRR con sus 9 dimensiones, el Bus Cognitivo y el SAN estan en la capa Core. No hay funcionalidad cientifica detras de un paywall.

Es open source?

Si y no — depende de que capa. MNEMOS tiene 4 niveles de licencia, cada uno con un proposito especifico:

CapaLicenciaQue incluyePrecio
Core AGPLv3Nota TecnicaAGPLv3 obliga a publicar el codigo fuente si modificas MNEMOS y lo ofreces como servicio (SaaS). Esto evita que alguien tome el core, lo modifique y lo venda como servicio cerrado sin contribuir de vuelta. Si solo lo usas internamente, no hay obligacion de publicar. 4 motores cognitivos, CRR, API REST, MCP Server, Bus Cognitivo, SAN, 3 agentes Gratis
SDK Apache 2.0Nota TecnicaApache 2.0 es una licencia permisiva que permite uso comercial sin restricciones. Se eligio para el SDK porque queremos que cualquier empresa pueda integrar MNEMOS sin obligacion de abrir su codigo. Maximiza adopcion del ecosistema. Librerias cliente, conectores, formato .mnemos, protocolo MCP Gratis
Pro Comercial (source available) Biometricos avanzados, comunicaciones multi-plataforma, avatares digitales, analytics enterprise Suscripcion (por definir)
Legacy Propietario Legados cognitivos, self-clone, family Legacy, professional Legacy, Cortex colectivo Premium (por definir)

Analogia: Piensa en PostgreSQL. El motor de base de datos es gratuito y open source. Empresas como Crunchy Data o Supabase ofrecen servicios gestionados de pago encima. MNEMOS sigue el mismo principio: la ciencia cognitiva es libre, los servicios premium se pagan.

Datos y Privacidad

Donde se almacenan mis datos?

MNEMOS es self-hosted por diseno. Tus datos viven en tu propia base de datos SQL ServerNota TecnicaActualmente MNEMOS usa SQL Server para desarrollo (disponible en el equipo de trabajo). La migracion a PostgreSQL esta planeada para produccion, donde se aprovecharian pgvector (busqueda vectorial O(log N) con HNSW) y tsvector (full-text search nativo con indices GiST)., en tu maquina o servidor. No hay telemetria, no hay servidor central, no hay "nube de MNEMOS".

En el futuro se planea una opcion cloud para quienes prefieran no gestionar infraestructura, pero sera opcional y con cifrado end-to-end. Tu siempre podras elegir self-hosted.

Principio 0 de MNEMOS: El sistema es custodio neutral de la memoria. No posee tus datos — tu los posees. Los archivos .md son la fuente de verdad del contenido, legibles con cualquier editor de texto. Sin vendor lock-in.

Quien puede acceder a mis datos?

Actualmente MNEMOS es single-userFuncionamiento InternoEn la version actual no hay autenticacion — es desarrollo local. Para produccion se planea JWT + ASP.NET Identity con soporte multi-tenant. Cada tenant tendra su propio esquema de base de datos con aislamiento completo. Los trust_levels (0-4) controlaran que datos son compartibles.. Solo tu accedes a tu instancia. Multi-tenant esta planeado para produccion con aislamiento completo por usuario.

Instalacion y Plataformas

Es facil de instalar?

Si. MNEMOS es una aplicacion .NET 8Nota Tecnica.NET 8 es un LTS (Long Term Support) de Microsoft con soporte hasta noviembre 2026. Es cross-platform (Windows, Linux, macOS) y tiene un runtime de alto rendimiento. Un solo comando compila y ejecuta el proyecto completo. estandar. Los requisitos minimos son:

1

.NET 8 SDK

Descarga gratuita de dotnet.microsoft.com. Incluye compilador, runtime y herramientas CLI.

2

SQL Server

SQL Server Express (gratuito) o Developer Edition. Futuro: PostgreSQL 16+ para produccion.

3

Un terminal

dotnet run --project src/Mnemos.Server y listo. Tambien se puede publicar como .exe standalone.

# Clonar el repositorio
git clone https://github.com/fcostales/mnemos.git
cd mnemos

# Compilar
dotnet build

# Aplicar migraciones
dotnet ef database update --project src/Mnemos.Infrastructure

# Ejecutar el servidor (http://localhost:5000)
dotnet run --project src/Mnemos.Server

# Ejecutar el MCP Server (para Claude Code/Desktop)
dotnet run --project src/Mnemos.McpServer

Hay demo disponible?

No todavia. Hay app movil?

No nativa. El frontend actual es BlazorNota TecnicaBlazor es el framework de UI de .NET que permite escribir interfaces web interactivas en C# en lugar de JavaScript. El frontend de MNEMOS usa Blazor con un tema oscuro personalizado y 5 paginas (Home, NewMemor, MemorDetail, Search, Graph). (web), que funciona en el navegador del movil. El plan a futuro es hacer la UI responsive y empaquetar como PWA (Progressive Web App), lo que permitiria instalarlo en el movil sin necesidad de app stores.

Funciona offline?

Si, con limitaciones. MNEMOS tiene un StubAIEngineCodigo C#El StubAIEngine genera embeddings locales usando bag-of-words + feature hashing cuando no hay conexion a APIs externas. Funcional pero con menor precision semantica que OpenAI.

public class StubAIEngine : IAIEngine
{
    // Genera embeddings locales sin API
    // Bag-of-words + feature hashing
    // Funcional: busqueda por keywords
    // Limitado: sin comprension semantica
}
que genera embeddings locales cuando la API de OpenAI no esta disponible. La busqueda funciona (por keywords y similitud basica), los 4 motores cognitivos funcionan, y todas las operaciones CRUD funcionan. Lo que pierdes sin conexion es la precision de los embeddings semanticos de OpenAI.

Empresa y Sostenibilidad

Hay una empresa detras de MNEMOS?

No todavia. MNEMOS es un proyecto personalContextoMNEMOS nacio de una pregunta simple: "Deberia usar Obsidian con Claude Code?" La investigacion derivo en 303 debates con agentes PhD, 12 filosofos como jurado, 101 ingenieros auditando, y 10.000 usuarios simulados. Lo que empezo como una herramienta personal se convirtio en un sistema cognitivo fundamentado en ciencia.«De 'deberia usar Obsidian?' a un sistema operativo cognitivo civilizacional.» de Felipe Costales, en fase activa de desarrollo. La vision a largo plazo contempla una fundacion (MNEMOS Foundation) con gobernanza constitucional, pero eso esta en fases futuras.

Cual es el modelo de sostenibilidad?

Open core: el Core es gratuito y siempre lo sera (AGPLv3 lo garantiza legalmente). Los ingresos vendran de las capas Pro y Legacy, que ofrecen funcionalidad avanzada para usuarios que necesitan biometricos, comunicaciones multi-plataforma, avatares digitales o legados cognitivos.

Analogia: Como Red Hat con Linux. El kernel es libre. El soporte, la certificacion y las herramientas enterprise se pagan. MNEMOS sigue el mismo modelo: la ciencia es libre, los servicios premium generan sostenibilidad.

Hay soporte?

Actualmente via GitHub IssuesNota TecnicaEl repositorio de MNEMOS usa GitHub para gestion de codigo, issues y pull requests. Las contribuciones siguen el flujo GitFlow (main + develop + feature branches) con conventional commits en espanol y squash merge a main. y comunidad. En fases futuras, la capa Pro incluira soporte prioritario con SLAs definidos.

Self-hosted o cloud?

Self-hosted como primera opcion y opcion por defecto. Tus datos, tu servidor, tu control total. Cloud sera una opcion futura para quienes no quieran gestionar infraestructura, pero nunca sera el unico camino. MNEMOS respeta el Principio 3: soberania del datoPrincipio FilosoficoEl Principio 3 de MNEMOS establece que el usuario es dueno absoluto de sus datos. El sistema es custodio, no propietario. Formato .mnemos abierto. Exportacion total en cualquier momento. Si MNEMOS desapareciera manana, tus archivos .md seguirian siendo legibles.«Tu memoria no deberia depender de que una empresa siga existiendo.».
